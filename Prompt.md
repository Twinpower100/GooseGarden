Ты senior full-stack engineer, product-minded frontend architect и solution designer. Твоя задача — создать полноценный production-ready проект премиального продающего сайта для ландшафтного дизайнера.

Это не просто “сайт с формой и галереей”. Это маркетинговый сайт, который должен:
- визуально выглядеть дорого;
- вызывать доверие;
- подталкивать клиента оставить заявку;
- хорошо работать на мобильных устройствах;
- иметь управляемый контент через админку;
- не зависеть от платных сервисов.

Работай как исполнитель, а не как консультант:
- не задавай уточняющих вопросов;
- не предлагай альтернативы;
- не оставляй TODO, FIXME, placeholders;
- не заменяй реальную логику моками;
- не используй платные библиотеки, платные UI kits, платные CMS, платные шаблоны, платные API и платные SaaS;
- не используй Tailwind Plus, Motion+, платные theme packs, платные gallery plugins, коммерческие шаблоны;
- не используй WordPress;
- не используй Django templates;
- не используй «голый» React без полноценной архитектуры;
- создай законченную кодовую базу, готовую к локальному запуску и дальнейшему деплою.

ОБЯЗАТЕЛЬНЫЙ СТЕК

Используй именно этот стек:

Frontend + app framework:
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- Motion for React для анимаций
- только современный React-based подход, без устаревших шаблонных решений

Backend / CMS / Admin:
- Payload CMS, встроенный в тот же Next.js проект
- Payload admin panel для управления контентом
- Payload localization для контентных полей
- PostgreSQL

Формы и интеграции:
- серверная обработка запросов в рамках Next.js / Payload
- SMTP email уведомления
- Telegram Bot API для опциональных уведомлений
- без платных внешних сервисов

Хранение медиа:
- локальное файловое хранилище для dev
- архитектура должна быть готова к переносу на external storage позже, но без обязательного подключения платных сервисов

ОБЩАЯ ЦЕЛЬ

Нужно сделать премиальный сайт ландшафтного дизайнера, который не выглядит как шаблон, не выглядит как “типичный CMS-сайт”, не выглядит как “корпоративный конструктор”, а воспринимается как дорогой визуальный digital-продукт для услуг высокого уровня.

Сайт должен делать акцент на:
- визуальную подачу работ;
- атмосферу;
- композицию;
- крупные фото;
- видео;
- спокойные, дорогие анимации;
- выразительную типографику;
- доверие;
- удобный контакт и быстрый путь к заявке.

КЛЮЧЕВОЙ ПРИОРИТЕТ

Главный приоритет — сильный фронтенд и премиальный визуальный результат.

Не делай “админку ради админки”.
Не делай “просто аккуратный сайт”.
Сделай сильный продающий маркетинговый сайт с ощущением high-end services.

- использовать безопасную реализацию autoplay через muted + playsinline + loop, с fallback;
- галереи и видео должны выглядеть сильно и современно, а не как дефолтный слайдер из шаблона.

Нужны:
- страница списка портфолио;
- страницы деталей проекта;
- превью избранных работ на главной;
- фильтрация/группировка, если это можно сделать без усложнения UX;
- акцент на визуальную подачу кейсов.

- русский

Архитектура должна позволять добавлять новые языки без переделки проекта.

Нужно реализовать:

- переключатель языка;
- перевод интерфейса;
- перевод контентных полей;
- SEO metadata по языкам;

Все пользовательские тексты интерфейса должны быть интернационализированы.

Все контентные сущности должны иметь RU и EN версии там, где это нужно:

- about;
- services;
- portfolio;
- contact texts;

- form labels;
- success/error messages;
- meta title / description.

3. СОЦИАЛЬНЫЕ ССЫЛКИ

Через админку должны настраиваться:
- Facebook
- Instagram
- Telegram

Требования:
- если ссылка отсутствует, не показывать иконку/элемент;
- ссылки должны отображаться в шапке, футере и в зоне контактов, если это уместно;
- визуально интегрировать их аккуратно, без ощущения “иконки ради иконок”.

4. ФОРМА ЗАПРОСА НА УСЛУГИ

На сайте должна быть полноценная форма запроса на услуги.

Поля:
- имя
- телефон
- email
- preferred_contact_method
- project_location
- service_type
- short_request_description
- optional_comment
- consent_to_personal_data_processing

Требования:
- клиентская валидация;
- серверная валидация;
- сохранение в БД;
- отображение заявок в админке;
- понятные сообщения об успехе и ошибке;
- защита от спама без платных решений;
- форма должна быть удобной на телефоне;
- форма должна не ломать премиальное ощущение сайта.

Защита от спама:
- honeypot;
- rate limiting;
- серверная валидация;
- антибот-логика без внешних платных captcha-сервисов.

Уведомления:
- опциональная отправка уведомления на email через SMTP;
- опциональная отправка уведомления в Telegram через Bot API;
- если уведомления не настроены, форма все равно работает и сохраняет запрос в БД.

5. КОНТАКТЫ

Через админку должны настраиваться:
- email
- Telegram
- WhatsApp

Нужно реализовать предзаполненные сообщения:
- email через mailto:
- Telegram через ссылку с заранее подставленным текстом
- WhatsApp через wa.me с заранее подставленным текстом

Требования:
- тексты шаблонов сообщений должны быть настраиваемыми;
- для RU и EN должны быть отдельные версии шаблонов;
- если контакт не заполнен, он не отображается;
- переход по контакту должен вести к готовому черновику сообщения нужному адресату.

6. MOBILE FIRST И PREMIUM UI

Mobile-first обязателен.
Но сайт должен быть сильным и на десктопе, а не просто “терпимым на телефоне”.

Требования к UI/UX:
- премиальная типографика;
- крупные блоки;
- выразительная визуальная иерархия;
- хорошие ритмы отступов;
- большие touch targets;
- отсутствие горизонтального скролла;
- высокая читаемость;
- сильный визуальный hero;
- чистый layout;
- дорогая цветовая подача;
- не шаблонная композиция;
- subtle motion;
- scroll-based reveal animations;
- анимации появления секций;
- мягкие hover/focus states;
- плавные, но не навязчивые переходы;
- аккуратная работа с изображениями и видео;
- никакой дешевой “карусельной” эстетики;
- никакого ощущения boilerplate.

Дизайн должен ощущаться как сайт дорогой творческой услуги, а не как сайт обычного подрядчика.

7. SEO И MARKETING BASELINE

- metadata для всех страниц;
- Open Graph;
- Twitter cards, если уместно;
- canonical;
- sitemap;
- robots.txt;
- человекочитаемые URL;

ДОПОЛНИТЕЛЬНОЕ УСИЛЕНИЕ САЙТА КАК ПРОДАЮЩЕГО

АНИМАЦИИ

Используй Motion for React.

Анимации должны быть:
- дорогими по ощущению;
- спокойными;
- современными;
- не мультяшными;
- не перегруженными;
- не мешающими UX;
- не ухудшающими performance.

Подходящие паттерны:
- reveal on scroll;
- fade + slight translate;
- stagger;
- parallax-like subtle movement;
- elegant transitions between sections;
- аккуратные hover animations.

Нельзя:
- кислотные эффекты;
- резкие bouncing анимации;
- бесконечная визуальная суета;
- тяжелые и дерганые переходы;
- дешевый wow-effect ради wow-effect.

ЧТО СЧИТАТЬ УСПЕШНЫМ РЕЗУЛЬТАТОМ

1. проект запускается локально;
2. есть админка;
3. через админку можно менять контент;
4. портфолио управляется через CMS;
5. есть RU и EN;
6. форма заявки сохраняет данные;
7. контакты открывают предзаполненные сообщения;
8. сайт выглядит премиально;
9. мобильная версия сильная;
10. нет платных зависимостей;
11. проект не выглядит как типовой шаблон;
12. кодовая база законченная и связная.
